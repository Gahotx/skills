# 周报生成器 (Weekly Report Generator)

从 Git 项目中提取 commit 记录，自动生成结构化的工作周报。

## 使用场景

当用户请求以下内容时激活此 skill：
- "生成周报"、"写周报"、"本周工作总结"
- "汇总这周的提交"、"总结代码变更"
- "生成工作报告"、"导出开发日志"

## 快速开始

### 交互流程

1. **确认日期范围**：默认最近5天（含今天），或选择最近一周（上一个完整工作周），用户可自定义
2. **确认项目列表**：默认当前目录，支持多项目
3. **选择格式类型**：按commit类型 或 按项目分类
4. **确认保存位置**：默认 `docs/weekly-report/`，相对路径时可选 gitignore
5. **生成周报**：提取 commit 记录并生成
6. **补充内容**：询问额外事项和下周计划
7. **保存文件**：按结束日期命名

### 示例对话

**用户**: 帮我生成本周的周报

**助手**: 好的，我来帮你生成周报。首先确认几个信息：

> **问题 1**: 请选择周报日期范围：
> - 默认：最近5天（2026.05.15 - 2026.05.19）
> - 最近一周：上一个完整工作周（2026.05.11 - 2026.05.15）
> - 自定义日期

**用户**: 最近一周

**助手**:
> **问题 2**: 项目路径：默认使用当前目录，还有其他项目需要加入周报吗？

**用户**: 加上 E:\Projects\ai-assistant

**助手**:
> **问题 3**: 周报格式选择：
> - 按commit类型分类（功能开发、Bug修复等）
> - 按项目分类（每个项目单独一节）

**用户**: 按项目分类

**助手**:
> **问题 4**: 保存位置：默认保存到 docs/weekly-report/ 目录，需要修改吗？

**用户**: 默认位置

**助手**:
> **问题 5**: 是否需要在 .gitignore 中忽略该目录？

**用户**: 需要加入gitignore

**助手**: 正在收集提交记录...

（生成周报内容）

**助手**: 已根据commit记录生成周报初稿：

> **问题 6**: 是否还有其他额外的工作事项需要补充到"其他"中？

**用户**: 这周还参加了一次技术评审会议

**助手**:
> **问题 7**: 是否需要写下周工作计划？

**用户**: 需要

（用户输入下周计划）

**助手**: 周报已保存到 docs/weekly-report/2026-05-16.md

## 注意事项

- 所有与用户的交互必须使用中文
- 如果某个项目在指定日期范围内没有 commit 记录，不要在周报中列出该项目
- 如果所有项目都没有 commit 记录，提示用户确认日期范围是否正确
- 日期格式统一使用 `YYYY-MM-DD`
- 周报标题中的日期格式为 `YYYY.MM.DD - YYYY.MM.DD`（如：2026.05.11 - 2026.05.15）
