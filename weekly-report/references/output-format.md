# 周报输出格式

**重要**：最终输出必须是 Markdown 格式。根据用户选择的格式类型，使用对应的模板。

保存 Markdown 文件后，需要额外输出一份纯文本版本方便用户复制到微信群等场景。

---

## 按项目分类格式

当用户选择"按项目分类"时，使用以下模板：

**重要提示**：按项目分类时，每个项目下直接列出工作内容，**不要**添加"功能开发"、"Bug修复"等子级标题。项目下的每条内容应该是对commit的精炼总结，而不是按类型分组。

```markdown
# 工作周报（{start_year}.{start_month}.{start_day} - {end_year}.{end_month}.{end_day}）

## 一、本周工作总结

### 1. {project_name_1}
- {commit_message_1}
- {commit_message_2}
- {commit_message_3}

### 2. {project_name_2}
- {commit_message_1}
- {commit_message_2}

### 3. 其他
- {extra_work_1}
- {extra_work_2}

## 二、下周工作计划

### 1. {project_name_1}
- {plan_item_1}
- {plan_item_2}

### 2. {project_name_2}
- {plan_item_1}

## 三、备注
> {notes}
```

**示例输出**：
```markdown

## 日期格式说明

周报标题中的日期格式：
- `{start_year}`: 开始日期的年份（4位数字）
- `{start_month}`: 开始日期的月份（2位数字，补零）
- `{start_day}`: 开始日期的日（2位数字，补零）
- `{end_year}`: 结束日期的年份（4位数字）
- `{end_month}`: 结束日期的月份（2位数字，补零）
- `{end_day}`: 结束日期的日（2位数字，补零）

**格式模板**：`工作周报（YYYY.MM.DD - YYYY.MM.DD）`

**示例**：
- 开始日期：2026-05-11 → `2026.05.11`
- 结束日期：2026-05-15 → `2026.05.15`
- 最终标题：`工作周报（2026.05.11 - 2026.05.15）`

---

## 文件命名规则

周报文件名使用结束日期：
```
{end_date}.md
```

**示例**：
- 结束日期：2026-05-15 → 文件名：`2026-05-15.md`

## 纯文本版本输出

保存 Markdown 文件后，需要额外输出一份纯文本版本供用户复制到微信群等场景。

**转换规则**：
1. 移除所有 `#` 标记（`#`、`##`、`###` 等）
2. 标题层级使用数字编号替代：
   - `# 工作周报` → `工作周报`
   - `## 一、本周工作总结` → `一、本周工作总结`
   - `### 1. 项目名称` → `1. 项目名称`
3. 保留 `-` 列表符号
4. 保留所有文字内容

**输出格式**：

周报已生成完成！保存路径：`{output_path}/{end_date}.md`

周报内容：

```
{纯文本周报内容}
```

**示例输出**：

周报已生成完成！保存路径：`C:/Users/HP/Desktop/周报/2026-05-15.md`

周报内容：

```
