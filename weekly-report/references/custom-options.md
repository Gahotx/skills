# 自定义选项

在生成周报前后，用户可以自定义以下内容。

---

## 生成前配置

### 1. 周报日期范围 ⭐ 必须确认

**询问时机**：开始生成周报时

**选项**：
- 默认：最近5天（含今天，往前推4天）
- 最近一周：上一个完整工作周（上周一到上周五）
- 自定义：用户指定开始日期和结束日期

**询问示例**：
> 请选择周报日期范围：
> - 默认：最近5天（2026.05.23 - 2026.05.27）
> - 最近一周：上一个完整工作周（2026.05.18 - 2026.05.22）
> - 自定义日期

**用户回答示例**：
- "默认" → 使用最近5天
- "最近一周" → 使用周一到周五
- "改成5月11日到5月17日" → 使用自定义日期

---

### 2. 项目列表 ⭐ 必须确认

**询问时机**：确认日期范围后

**选项**：
- 默认：当前工作目录
- 自定义：用户输入一个或多个项目路径

**询问示例**：
> 项目路径：默认使用当前目录，还有其他项目需要加入周报吗？

**用户回答示例**：
- "没有了" → 仅使用当前目录
- "加上 E:\Projects\ai-assistant" → 添加单个项目
- "E:/Projects/frontend, E:/Projects/backend, E:/Projects/admin" → 添加多个项目

---

### 3. 周报格式 ⭐ 必须确认

**询问时机**：确认项目列表后

**选项**：
- 默认：按项目分类（每个项目单独一节）
- 按commit类型分类：功能开发、Bug修复、代码重构等

**询问示例**：
> 周报格式选择：
> - 按commit类型分类（功能开发、Bug修复等）
> - 按项目分类（每个项目单独一节）

**用户回答示例**：
- "按项目分类" → 使用按项目分类格式
- "按类型分类" → 使用按类型分类格式

---

### 4. 保存位置 ⭐ 必须确认

**询问时机**：确认格式后

**选项**：
- 默认：桌面的 `周报/` 目录（如 `C:/Users/用户名/Desktop/周报/`）
- 自定义：用户指定其他保存路径

**询问示例**：
> 保存位置：默认保存到 `桌面/周报/` 目录，需要修改吗？

**用户回答示例**：
- "默认位置" → 保存到默认位置
- "保存到 E:\工作文档\周报" → 保存到自定义位置

---

## 生成后补充

### 5. 额外工作事项 ⭐ 必须询问

**询问时机**：生成周报内容后

**说明**：
有些工作任务可能不会在 commit 记录中体现，例如：
- 参加会议
- 代码评审
- 技术讨论
- 文档编写
- 问题排查

**询问示例**：
> 是否还有其他额外的工作事项需要补充到"其他"中？

**用户回答示例**：
- "没有了" → 不添加额外内容
- "这周还参加了一次技术评审会议" → 添加到"其他"分类

---

### 6. 下周工作计划 ⭐ 必须询问

**询问时机**：确认额外事项后

**选项**：
- 不需要：留空或写"暂无"
- 需要：用户输入下周工作计划

**询问示例**：
> 是否需要写下周工作计划？

**用户回答示例**：
- "不需要" → 下周计划部分留空
- "需要" → 进入计划输入流程

**计划输入询问示例**：
> 请输入下周的工作计划（按项目或按条目输入均可）：

**用户输入示例**（按项目）：
```
康养管理端
- 0522版本开发与上线
- 0605版本评审

稳态AI助理
- 0522版本开发与上线
- 消息中心功能优化
```

**用户输入示例**（按条目）：
```
- 完成0522版本开发
- 参加0605版本评审
- 优化消息中心功能
```

---

## 配置总结

| 配置项 | 是否必须 | 默认值 | 说明 |
|--------|----------|--------|------|
| 日期范围 | 是 | 最近5天 | 含今天，往前推4天 |
| 项目列表 | 是 | 当前目录 | 支持多项目，逗号分隔 |
| 周报格式 | 是 | 按项目分类 | 或按commit类型分类 |
| 保存位置 | 是 | 桌面/周报/ | 支持自定义路径 |
| 额外事项 | 是（询问） | 空 | commit记录未体现的工作 |
| 下周计划 | 是（询问） | 空 | 下周工作安排 |
